## 第0阶段：环境搭建（1天）

### 目标
把项目跑起来，亲眼看到客户端能注册、登录、聊天。

### 步骤

**1. 安装基础依赖**

```bash
# 编译器与构建工具
sudo apt install -y g++ cmake make git

# 第三方库
sudo apt install -y libmysqlclient-dev libhiredis-dev libssl-dev libboost-dev libcurl4-openssl-dev
```

**2. 编译安装 Muduo**

```bash
git clone --depth 1 https://github.com/chenshuo/muduo /tmp/muduo
cd /tmp/muduo
# 修复 Boost 兼容性警告
sed -i 's/-Werror//' CMakeLists.txt
sed -i 's/-Wold-style-cast/-Wno-old-style-cast/' CMakeLists.txt
mkdir build && cd build
cmake -DCMAKE_INSTALL_PREFIX=/usr/local ..
make muduo_net muduo_base -j4
sudo make install
```

**3. 安装 MySQL 和 Redis**

```bash
sudo apt install -y mysql-server redis-server
sudo systemctl start mysql redis-server
```

**4. 初始化数据库**

```bash
# 创建数据库和表
sudo mysql < scripts/init.sql
# 验证
mysql -u root -e "USE chat; SHOW TABLES;"
```

**5. 编译运行**

```bash
cd ~/chatserver_v2
cd build && cmake .. && make -j4
# 启动服务端
./bin/ChatServer config/server.conf
# 另开终端，启动客户端
./bin/ChatClient 127.0.0.1 6000
```

**6. 亲自走一遍流程**

```
# 注册两个用户
[终端1] → 注册 用户A
[终端2] → 注册 用户B

# 用户A 加 用户B 为好友，发消息
[终端1] → 登录A → addfriend:2 → chat:2:你好

# 用户B 收到消息
[终端2] → 登录B → 看到 "你好"
```

### 这一阶段你要搞懂的问题
- Muduo 是什么？为什么不用手写 socket？
- MySQL、Redis 在这个项目里各有几个角色？
- `CMakeLists.txt` 里的 `target_link_libraries` 链接了哪些库？每个库是干什么的？

## 第1阶段：读懂代码（2-3天）

### 核心原则
**先整体后局部，先上层后底层。** 不要一头扎进某个 .cpp 看几百行。

### Day 1：理解消息怎么流动

按一次"用户A登录 → 加好友 → 给B发消息"的完整链路，从客户端到服务端逐层跟踪：

```
第一步：看客户端怎么发的
  打开 src/client/main.cpp
  ├── main() 函数：连接服务器 → 显示菜单
  ├── case 1 (login): 构造 JSON → send()
  ├── readTaskHandler(): recv() → 解析 JSON → doLoginResponse()
  └── chat(): 发送 chat:friendid:message 格式消息

第二步：看服务端怎么收的
  打开 src/server/chatserver.cpp
  └── onMessage(): buffer → JSON parse → get msgid → 查表回调

第三步：看业务层怎么处理的
  打开 src/server/chatservice.cpp
  ├── 构造函数：_msgHandlerMap 注册了哪些 handler？
  ├── login(): 验证密码 → 记录连接 → 拉离线消息/好友/群组 → 回复
  ├── oneChat(): 查本机 → 查 Redis → 存离线 → 回复 ACK
  └── getHandler(): O(1) 查表

第四步：看数据怎么存的
  打开 src/server/model/usermodel.cpp
  └── insert(): hashPassword() → escape() → snprintf() → mysql.update()

第五步：看跨服务器怎么通信的
  打开 src/server/redis/redis.cpp
  └── publish()/subscribe()/observer_channel_message()
```

### Day 2：理解每个模块的职责

按项目分层逐一梳理：

| 分层 | 核心文件 | 关键问题 |
|------|---------|---------|
| 网络层 | `chatserver.hpp/cpp` | Muduo 怎么注册回调？`setThreadNum(4)` 什么意思？ |
| 业务层 | `chatservice.hpp/cpp` | 单例怎么实现的？msgid→handler 映射表怎么工作？ |
| 数据层 | `model/*.cpp` | 每个 Model 封装了哪些 SQL？多表 JOIN 怎么写？ |
| DB 封装 | `db/db.cpp` | `mysql_real_escape_string` 怎么防注入？ |
| 连接池 | `db/connectionpool.cpp` | `condition_variable` 怎么配合 `queue` 实现阻塞获取？ |
| Redis | `redis/redis.cpp` | 为什么需要两个 redisContext？subscribe 为什么开独立线程？ |
| 配置 | `config.cpp` | JSON 配置文件怎么加载？怎么用默认值兜底？ |
| 密码 | `password_utils.cpp` | PBKDF2 三个参数（salt/迭代次数/key长度）各是什么含义？ |

### Day 3：默写核心流程

合上代码，在纸上画出以下流程图：

1. **消息分发流程**：客户端 send → onMessage → JSON parse → getHandler → handler 回调
2. **一对一聊天分支**：本机在线 / Redis在线 / 离线存储 三条路径
3. **心跳检测流程**：活跃时间更新 → 定时检查 → 超时断开 → 清理状态
4. **登录响应包含哪些数据**：用户信息 + 离线消息 + 好友列表 + 群组列表

## 第2阶段：模块拆解练习（3-5天）

### 练习1：去掉某一个组件，看哪里报错

用 `#if 0` ... `#endif` 注释掉：
- 连接池的 `init()` 调用 → 看哪里链接失败
- `_msgHandlerMap` 中注释掉一个 handler → 看对应消息返回什么
- Redis `connect()` 中的 `thread t(...)` → 看跨服消息是否还能送达

**目的**：理解模块间的依赖关系。

### 练习2：加一个新消息类型

比如加一个 "用户改名" 功能：

1. `public.hpp`：加 `CHANGE_NAME_MSG = 14`
2. `chatservice.hpp`：声明 `void changeName(conn, js, time)`
3. `chatservice.cpp`：注册 handler + 实现（校验 → 更新 MySQL → 回复 ACK）
4. `usermodel.hpp/cpp`：加 `bool updateName(int id, string newName)`
5. 客户端：加 `changename:newname` 命令

### 练习3：给一个模块写单元测试

从最简单的 `User` 类开始：

```cpp
// 不需要 MySQL，纯逻辑测试
TEST(UserTest, DefaultState)
{
    User u;
    EXPECT_EQ(u.getId(), -1);
    EXPECT_EQ(u.getState(), "offline");
}

TEST(PasswordTest, HashAndVerify)
{
    string hash = hashPassword("mypassword");
    EXPECT_TRUE(verifyPassword("mypassword", hash));
    EXPECT_FALSE(verifyPassword("wrong", hash));
    // 同一个密码两次 hash 结果不同（盐不同）
    string hash2 = hashPassword("mypassword");
    EXPECT_NE(hash, hash2);
}
```

### 练习4：性能对比实验

1. 把连接池大小从 8 改成 1
2. 写脚本并发创建 100 个连接同时登录
3. 观察响应延迟变化
4. 把 `shared_mutex` 改回 `mutex`，重复实验
5. 把 Redis 状态缓存注释掉（回退到每次查 MySQL），对比延迟

### 每个阶段的检查点

**阶段1 安全加固：**
- [ ] 密码从明文变成哈希了吗？（检查数据库 password 列长度是否为 97）
- [ ] 输入含单引号的用户名还能正常注册吗？
- [ ] 1001 个字符的消息会被拒绝吗？

**阶段2 性能优化：**
- [ ] 连接池初始化后 `_pool.size()` 是否为 8？
- [ ] `_userConnMap` 的锁从 `mutex` 变成 `shared_mutex` 了吗？
- [ ] 发消息时 MySQL 查询次数减少了吗？（开 general_log 对比）

**阶段3 可靠性：**
- [ ] 客户端 kill -9 后 90 秒内服务端是否检测到并清理？
- [ ] 同一条消息发两次，服务端返回 "duplicate" 了吗？

**阶段4 工程化：**
- [ ] 改 `server.conf` 里的端口号，重启后是否生效？
- [ ] `docker-compose up` 能否一键启动？

## 常见踩坑记录

| 问题 | 原因 | 解决 |
|------|------|------|
| `fatal error: muduo/net/TcpServer.h` | Muduo 未安装或路径不对 | 检查 `/usr/local/include/muduo/` 是否有文件 |
| `undefined reference to ZSTD_*` | 缺少 zstd 链接 | 加 `-lzstd` 或链接 `.so` 完整路径 |
| mysql 连接被拒绝 | 密码不对或无权限 | `sudo mysql -e "ALTER USER 'root'@'localhost' IDENTIFIED WITH mysql_native_password BY '123456';"` |
| 客户端连接超时 | 服务端没启动或端口被防火墙挡 | `ss -tlnp \| grep 6000` 检查监听 |
| Redis 连不上 | redis-server 没启动 | `redis-server --daemonize yes` |
| `register error!` | 用户名已被注册（name 字段 UNIQUE） | 换一个用户名 |
| Docker 构建时 Muduo 编译失败 | CMake -Werror 导致 Boost 兼容性告警变错误 | 已在 Dockerfile 中 `sed` 处理 |
| `sem_wait` 死等 | 服务端没返回响应或子线程没处理 ACK | 检查服务端是否正常运行，消息类型是否正确 |
