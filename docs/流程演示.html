// 从 js 里提取业务字段 int toid = js["toid"]; // 目标用户ID int fromid = js["id"]; // 发送者ID string uuid = js["msg_uuid"]; // 去重key ────────────────────────────── // 用 conn 回复发送者 conn→send(ack.dump()) ← conn 原路返回 // 用 time 更新活跃时间 _lastActiveMap[fromid] = time // 用 toid 查路由, 用 js 发消息 → 见下方三级路由分支
oneChat() 内部三级路由 — 根据 toid 决定投递路径
路径A: 本机在线
shared_lock(_connMutex) _userConnMap.find(toid) → 命中! ────────────────────── it→second→send(js.dump()) // 直接推送 conn→send("ACK ok") // 回复发送者 ────────────────────── 参数流: toid→查map→TcpConnection→send(原js)
路径B: 跨服务器在线
_userConnMap.find(toid) → 未命中 _redis.getUserState(toid) → "online" ────────────────────── _redis.publish(toid, js.dump()) // 发布到Redis conn→send("ACK ok") ────────────────────── 参数流: toid→publish(channel, msg)→目标服务器observer线程
路径C: 离线存储
_redis.getUserState(toid) → nil _userModel.query(toid) → state="offline" ────────────────────── _offlineMsgModel.insert(toid, js.dump()) conn→send("ACK offline") ────────────────────── 参数流: toid+json→INSERT offlinemessage→等登录拉取

▼ Redis 层 (三种用途) — src/server/redis/redis.cpp
① Pub/Sub 消息总线
// 两个独立 redisContext _publish_context → PUBLISH channel msg _subcribe_context → SUBSCRIBE channel ────────────────────── // 独立线程阻塞监听 observer_channel_message(): redisGetReply(_subcribe) → _notify_handler(userid, msg) // 回调到业务层 → handleRedisSubscribeMessage
② K-V 状态缓存
setUserState(userid, "online") → SET user:1:state online ────────────────────── getUserState(userid) → GET user:1:state → "online"|"" ────────────────────── delUserState(userid) → DEL
③ SETNX 消息去重
setnxWithExpire(key, 3600) → SET msg:1_xxx 1 NX EX 3600 ────────────────────── 返回 true → 首次, 处理消息 返回 false → 重复, 仅回 ACK

▼ 全部 13 种消息类型 & 参数签名
// 所有 handler 签名统一: void handler(const TcpConnectionPtr& conn, json& js, Timestamp time) // 通过 _msgHandlerMap[msgid] = bind(&ChatService::函数名, this, _1, _2, _3) 注册 ──────────────────────────────────────────────────────────────────────────────────────────────── msgid 1 LOGIN_MSG login js{"id":int, "password":string} → verifyPassword() → subscribe() → 拉离线/好友/群组 msgid 2 LOGIN_MSG_ACK - response{"errno":0/1/2, "name", "friends", ...} → 客户端处理 msgid 3 LOGINOUT_MSG loginout js{"id":int} → 清理连接 + Redis + MySQL msgid 4 REG_MSG reg js{"name":string, "password":string} → hashPassword() → INSERT msgid 5 REG_MSG_ACK - response{"errno":0, "id":int} → 客户端打印 msgid 6 ONE_CHAT_MSG oneChat js{"id","toid","msg","msg_uuid"} → 三级路由 + ACK msgid 7 ADD_FRIEND_MSG addFriend js{"id":int, "friendid":int} → INSERT friend msgid 8 CREATE_GROUP_MSG createGroup js{"id":int, "groupname", "groupdesc"} → INSERT allgroup + groupuser msgid 9 ADD_GROUP_MSG addGroup js{"id":int, "groupid":int} → INSERT groupuser msgid 10 GROUP_CHAT_MSG groupChat js{"id","groupid","msg","msg_uuid"} → 查成员→遍历三级路由→ACK msgid 11 PING_MSG ping js{"id":int} → 更新活跃时间→回PONG msgid 12 PONG_MSG - response{} → 客户端静默 msgid 13 MSG_ACK - response{"msg_uuid", "status":"ok"|"offline"|"duplicate"} → 客户端静默
