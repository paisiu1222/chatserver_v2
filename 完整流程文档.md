# 集群聊天服务器 — 完整流程文档

> 覆盖每一个功能的客户端→服务端→数据库→返回 全链路

---

## 0. 系统启动流程

### 0.1 服务端启动

```
./bin/ChatServer config/server.conf
```

**main.cpp 执行序列：**

```
1. Config::instance()->load("config/server.conf")
   └── config.cpp: 读取 JSON, 解析 server/mysql/redis/log 配置段

2. 解析配置:
   ip   = cfg->serverIp()      → 默认 "0.0.0.0"
   port = cfg->serverPort()    → 默认 6000

3. signal(SIGINT, resetHandler)   ← 注册 Ctrl+C 处理

4. EventLoop loop;                ← Muduo 主事件循环
   InetAddress addr(ip, port)
   ChatServer server(&loop, addr, "ChatServer")
   └── chatserver.cpp 构造函数:
       ├── _server.setConnectionCallback(onConnection)
       ├── _server.setMessageCallback(onMessage)
       └── _server.setThreadNum(4)   ← 1 main + 3 sub reactor

5. server.start()  → TcpServer::start()  → 开始监听

6. loop.runEvery(30.0, checkIdleConnections)  ← 心跳定时器

7. LOG_INFO "ChatServer started on 0.0.0.0:6000"

8. loop.loop()   ← 进入事件循环, 阻塞
```

**ConnectionPool 初始化（延迟到首次数据库操作）：**

```
第一次 MySQL::connect() 调用时:
  → ConnectionPool::instance()->init(host, port, user, pwd, db, 8)
  → 循环 8 次:
      mysql_init() → mysql_real_connect() → mysql_query("set names gbk")
      → push 到 _pool 队列
```

**Redis 初始化（ChatService 构造函数）：**

```
ChatService::ChatService():
  → _redis.connect()
    ├── redisConnect("127.0.0.1", 6379) → _publish_context
    ├── redisConnect("127.0.0.1", 6379) → _subcribe_context
    └── thread(observer_channel_message).detach()  ← 独立线程阻塞监听
  → _redis.init_notify_handler(handleRedisSubscribeMessage)
```

### 0.2 客户端启动

```
./bin/ChatClient 127.0.0.1 6000
```

**main.cpp 执行序列：**

```
1. socket(AF_INET, SOCK_STREAM, 0)  → clientfd
2. connect(clientfd, 127.0.0.1:6000)
3. sem_init(&rwsem, 0, 0)           ← 信号量初始值 0
4. thread(readTaskHandler, clientfd).detach()  ← 子线程启动, 阻塞 recv
5. 主线程进入死循环:
   └── 显示菜单 (1.login / 2.register / 3.quit)
```

## 1. 注册流程

### 用户操作
```
客户端菜单选择 2
输入: username → "测试用户"
输入: userpassword → "test123"
```

### 客户端 (main.cpp)

```
case 2: // register
  cin.getline(name, 50)   → "测试用户"
  cin.getline(pwd, 50)    → "test123"

  构造 JSON:
  {
    "msgid": 4,           // REG_MSG
    "name": "测试用户",
    "password": "test123"
  }

  send(clientfd, json_str, len, 0)  ──────TCP────→
  sem_wait(&rwsem)                    ← 阻塞等待子线程通知
```

### 服务端接收 (chatserver.cpp)

```
onMessage(conn, buffer, time):
  buf = buffer->retrieveAllAsString()
  js = json::parse(buf)                    → {"msgid":4, "name":"测试用户", "password":"test123"}
  msgHandler = ChatService::instance()->getHandler(4)  → reg 函数
  msgHandler(conn, js, time)
```

### 服务端处理 (chatservice.cpp)

```
ChatService::reg(conn, js, time):

  1. 输入校验
     name = "测试用户"    isValidName()     → true (长度 2-20)
     pwd  = "test123"     isValidPassword() → true (长度 6-100)
     不通过 → 返回 REG_MSG_ACK {errno=1, errmsg="invalid..."}

  2. 组装 User 对象
     user.setName("测试用户")
     user.setPwd("test123")

  3. _userModel.insert(user)
     └── usermodel.cpp insert():
         ├── hashPassword("test123")
         │   └── password_utils.cpp:
         │       RAND_bytes(salt, 16)           → 随机 16 字节盐
         │       PKCS5_PBKDF2_HMAC(..., 100000, SHA256, 32, key)
         │       return hex(salt) + ":" + hex(key)  → 97 字符
         │
         ├── ConnectionPool::acquire() → 阻塞获取 MYSQL* 连接
         ├── mysql.escape("测试用户")  → 转义特殊字符
         ├── snprintf(sql, "insert into user(name,password,state) values('测试用户','<97字符哈希>','offline')")
         ├── mysql.update(sql)        → INSERT 执行
         └── user.setId(mysql_insert_id(conn))  → 获取自增 ID
```

### 服务端响应

```
插入成功:
  response = {
    "msgid": 5,           // REG_MSG_ACK
    "errno": 0,            // 0=成功
    "id": 1                // 新用户 ID
  }
  conn->send(response.dump())  ──────TCP────→
```

### 客户端接收

```
readTaskHandler 子线程:
  recv(clientfd, buffer, 1024)
  json::parse(buffer) → msgtype = 5 (REG_MSG_ACK)
  
  doRegResponse(js):
    errno=0 → "name register success, userid is 1, do not forget it!"

  sem_post(&rwsem)  → 通知主线程继续

主线程:
  sem_wait 返回 → 回到主菜单循环
```

### 数据库变化

```sql
-- user 表新增一行:
INSERT INTO user(name, password, state)
VALUES('测试用户', 'a1b2c3...97chars...f0e1', 'offline');
-- id 自动分配为 1
```

### 用户操作
```
客户端菜单选择 1

```

### 客户端 (main.cpp)

```
case 1: // login
  cin >> id          → 1
  cin.getline(pwd, 50) → "test123"

  构造 JSON:
  {
    "msgid": 1,       // LOGIN_MSG
    "id": 1,
    "password": "test123"
  }

  g_isLoginSuccess = false
  send(clientfd, json_str, len, 0)  ──────TCP────→
  sem_wait(&rwsem)                  ← 阻塞等待
```

### 服务端接收 (chatserver.cpp)

```
onMessage(conn, buffer, time):
  buf → json::parse → js = {"msgid":1, "id":1, "password":"test123"}
  getHandler(1) → login
  login(conn, js, time)
```

### 服务端处理 (chatservice.cpp)

```
ChatService::login(conn, js, time):

  1. 输入校验
     id=1, pwd="test123" → isValidPassword → true

  2. 查数据库
     _userModel.query(1)
     └── usermodel.cpp:
         snprintf(sql, "select * from user where id = 1")
         mysql.query(sql) → mysql_fetch_row
         返回 User {id=1, name="测试用户", password="a1b2...hash...", state="offline"}

  3. 密码验证
     verifyPassword("test123", "a1b2...hash...")
     └── password_utils.cpp:
         提取 salt 和 expected_key
         PKCS5_PBKDF2_HMAC("test123", salt, 100000, SHA256, 32, key)
         CRYPTO_memcmp(key, expected_key, 32) == 0  → true ✓

  4. 检查重复登录
     user.getState() == "online"?  → 否, 当前是 "offline"

  5. 登录成功 — 记录连接
     unique_lock(_connMutex):
       _userConnMap[1] = conn        ← 记录 TCP 连接
       _lastActiveMap[1] = time      ← 初始化心跳时间

  6. Redis 订阅
     _redis.subscribe(1)             ← 订阅 channel 1 (本用户的跨服消息入口)
     └── redis.cpp:
         redisAppendCommand(_subcribe_context, "SUBSCRIBE 1")
         redisBufferWrite(...)

  7. 更新状态
     user.setState("online")
     _userModel.updateState(user)    → UPDATE user SET state='online' WHERE id=1
     _redis.setUserState(1, "online") → SET user:1:state online

  8. 拉取离线消息
     _offlineMsgModel.query(1)
     └── SELECT message FROM offlinemessage WHERE userid=1
     → 如果有, 放入 response["offlinemsg"], 然后 DELETE FROM offlinemessage WHERE userid=1

  9. 拉取好友列表
     _friendModel.query(1)
     └── SELECT a.id,a.name,a.state FROM user a
         INNER JOIN friend b ON b.friendid=a.id WHERE b.userid=1
     → 每个好友转为 JSON {id, name, state}, 放入 response["friends"]

  10. 拉取群组列表
      _groupModel.queryGroups(1)
      └── 先查所属群组:
          SELECT a.id,a.groupname,a.groupdesc FROM allgroup a
          INNER JOIN groupuser b ON a.id=b.groupid WHERE b.userid=1
      └── 再查每个群组的成员:
          SELECT a.id,a.name,a.state,b.grouprole FROM user a
          INNER JOIN groupuser b ON b.userid=a.id WHERE b.groupid=?
      → 组装为 [{id, groupname, groupdesc, users:[{id,name,state,role}]}]
      → 放入 response["groups"]
```

### 服务端响应

```json
{
  "msgid": 2,
  "errno": 0,
  "id": 1,
  "name": "测试用户",
  "offlinemsg": ["...", "..."],    // 有离线消息时才出现
  "friends": ["{\"id\":2,\"name\":\"张三\",\"state\":\"online\"}"],
  "groups": ["{\"id\":1,\"groupname\":\"技术群\",\"groupdesc\":\"技术交流\",\"users\":[...]}"]
}
```

### 客户端接收

```
readTaskHandler:
  recv → json::parse → msgtype = 2 (LOGIN_MSG_ACK)

  doLoginResponse(js):
    errno=0 → 登录成功
    ├── g_currentUser.setId(1), setName("测试用户")
    ├── 解析 friends → g_currentUserFriendList
    ├── 解析 groups → g_currentUserGroupList
    ├── showCurrentUserData() → 打印用户信息/好友/群组
    ├── 解析 offlinemsg → 逐条打印离线消息
    └── g_isLoginSuccess = true

  sem_post(&rwsem) → 通知主线程

主线程:
  sem_wait 返回
  g_isLoginSuccess=true → mainMenu(clientfd) → 进入聊天菜单
```

### 数据库/Redis 变化

```
MySQL:
  UPDATE user SET state='online' WHERE id=1

Redis:
  SUBSCRIBE channel 1  (由 observer_channel_message 线程监听)
  SET user:1:state online
```

## 3. 一对一聊天流程

### 场景：用户1 给用户2 发消息

### 3.1 目标用户在本机在线

**用户操作：** `chat:2:你好世界`

**客户端 (main.cpp)：**

```
chat(clientfd, "2:你好世界"):
  解析 friendid=2, message="你好世界"

  构造 JSON:
  {
    "msgid": 6,         // ONE_CHAT_MSG
    "id": 1,
    "name": "测试用户",
    "toid": 2,
    "msg": "你好世界",
    "time": "2026-05-13 15:30:00",
    "msg_uuid": "1_1747123800"
  }
  send() ──────TCP────→
```

**服务端 (chatservice.cpp) — oneChat：**

```
1. 输入校验
   toid=2 (>0), msg "你好世界" (非空 ≤5000) → 通过

2. 消息去重
   msgUuid = "1_1747123800"
   _redis.setnxWithExpire("msg:1_1747123800", 3600)
   └── Redis: SET msg:1_1747123800 1 NX EX 3600
       返回 OK → 首次, 继续处理
       (如果已存在 → 返回 "duplicate" ACK, 结束)

3. 更新活跃时间
   unique_lock: _lastActiveMap[1] = time

4. 查本机在线
   shared_lock(_connMutex):
     _userConnMap.find(2) → 找到了!
     it->second->send(js.dump())   ← 直接转发原消息给用户2

5. 回复发送者 ACK
   conn->send({
     "msgid": 13,        // MSG_ACK
     "msg_uuid": "1_1747123800",
     "status": "ok"
   })
```

**用户2 的客户端接收：**

```
readTaskHandler:
  recv → msgtype=6 (ONE_CHAT_MSG)
  → 打印: "2026-05-13 15:30:00 [1]测试用户 said: 你好世界"
```

**用户1 的客户端接收 ACK：**

```
readTaskHandler:
  recv → msgtype=13 (MSG_ACK)
  → 静默处理 (continue)
```

---

### 3.2 目标用户在集群另一台服务器在线

```
用户1 在 ServerA, 用户2 在 ServerB (通过 Nginx 分配)

ServerA 的 oneChat:

  1-3: (同上: 校验 → 去重 → 更新活跃时间)

  4. 查本机在线
     shared_lock: _userConnMap.find(2) → 没找到 (不在本机)

  5. 查 Redis 状态缓存
     _redis.getUserState(2)
     └── GET user:2:state → "online"  ← 缓存命中!

  6. Redis 发布到目标 channel
     _redis.publish(2, json_msg)
     └── PUBLISH 2 <json_msg>

  7. 回复 ACK {status: "ok"} 给发送者

───────────────────────────────────────

ServerB 的 observer_channel_message 线程:

  redisGetReply(_subcribe_context) → 收到订阅消息
  └── reply = [message, "2", "<json_msg>"]
  └── _notify_message_handler(2, "<json_msg>")
      └── ChatService::handleRedisSubscribeMessage(2, msg):

          shared_lock(_connMutex):
            _userConnMap.find(2) → 找到了!
            → it->second->send(msg)   ← 转发给 ServerB 上的用户2

  用户2 客户端 → 打印消息
```

---

### 3.3 目标用户离线

```
oneChat 继续:

  4. 查本机 → 没找到
  5. 查 Redis → GET user:2:state → nil (缓存未命中)
     → 回源 MySQL:
       _userModel.query(2) → state = "offline"
     → toState = "offline"
  
  6. 存储离线消息
     _offlineMsgModel.insert(2, json_msg)
     └── INSERT INTO offlinemessage VALUES(2, '<json>')

  7. 回复 ACK
     conn->send({
       "msgid": 13,
       "msg_uuid": "1_1747123800",
       "status": "offline"       ← 告知发送者消息已离线存储
     })
```

用户2 下次登录时 → 拉取离线消息 → 看到这条消息 → 删除离线记录。

## 7. 群聊流程

### 用户操作：`groupchat:1:大家好，欢迎加入`

### 客户端

```
groupchat(clientfd, "1:大家好，欢迎加入"):
  解析 groupid=1, message="大家好，欢迎加入"

  构造 JSON:
  {
    "msgid": 10,        // GROUP_CHAT_MSG
    "id": 1,
    "name": "测试用户",
    "groupid": 1,
    "msg": "大家好，欢迎加入",
    "time": "2026-05-13 15:35:00",
    "msg_uuid": "1_1747124100"
  }
  send() ──────TCP────→
```

### 服务端

```
ChatService::groupChat(conn, js, time):

  1. 输入校验: groupid=1 (>0), msg 非空 ≤5000 → 通过

  2. 消息去重
     _redis.setnxWithExpire("msg:1_1747124100")
     → 首次, 继续

  3. 更新活跃时间
     unique_lock: _lastActiveMap[1] = time

  4. 查询群组成员 (除发送者本人)
     _groupModel.queryGroupUsers(1, 1)
     └── SELECT userid FROM groupuser WHERE groupid=1 AND userid!=1
     → 返回 [2, 3]  (群内还有用户2和用户3)

  5. 遍历每个成员, 发送消息:

     用户2:
     shared_lock(_connMutex):
       _userConnMap.find(2) → 找到了 (本机在线)
       → it->second->send(js.dump())   ← 直接推送

     用户3:
       _userConnMap.find(3) → 没找到
       _redis.getUserState(3) → "online"  (缓存命中)
       → _redis.publish(3, js.dump())  ← 跨服发布

     用户4 (假设离线):
       _redis.getUserState(4) → nil
       → _userModel.query(4) → state="offline"
       → _offlineMsgModel.insert(4, js.dump())  ← 离线存储

  6. 回复发送者 ACK
     conn->send({"msgid":13, "msg_uuid":"1_1747124100", "status":"ok"})
```

### 群聊路由策略总结

```
群聊消息 → 获取群成员列表(id ≠ 发送者)
  └── 遍历每个成员:
      ├── 本机 _userConnMap 有 → 直接 send() (TcpConnection 推送)
      ├── Redis 状态缓存命中 "online" → publish(channel=userid) (跨服)
      └── 都不在线 → offlineMsgModel.insert() (写 MySQL 等下次登录拉取)
```

## 9. 客户端异常断开流程

### 触发：客户端进程被 kill、网络断开

### 服务端 (chatserver.cpp)

```
Muduo 底层检测到 TCP 连接断开
  → onConnection(conn) 回调:
      if (!conn->connected())  // 连接断开
        → ChatService::instance()->clientCloseException(conn)
        → conn->shutdown()
```

### 服务端 (chatservice.cpp)

```
ChatService::clientCloseException(conn):

  1. 反向查找用户
     unique_lock(_connMutex):
       遍历 _userConnMap:
         找到 it->second == conn 的条目
         user.setId(it->first)    → 得到 userid
         _userConnMap.erase(it)   → 移除连接

  2. Redis 清理
     _redis.unsubscribe(user.getId())
     _redis.delUserState(user.getId())

  3. 更新数据库
     if (user.getId() != -1):
       user.setState("offline")
       _userModel.updateState(user)
       → UPDATE user SET state='offline' WHERE id=?
```

## 10. 心跳机制

### 10.1 服务端定时器

```
// main.cpp
loop.runEvery(30.0, []() {
    ChatService::instance()->checkIdleConnections();
});
```

### 10.2 活跃时间更新时机

每次以下操作都会更新 `_lastActiveMap[userid]`：

| 操作 | 更新的 userid | 代码位置 |
|------|-------------|---------|
| 登录成功 | 登录者本人 | login() L101 |
| 发送一对一消息 | 发送者 | oneChat() L302 |
| 发送群聊消息 | 发送者 | groupChat() L414 |
| 收到 PING | PING 发送者 | ping() L478 |

### 10.3 空闲连接检测

```
ChatService::checkIdleConnections():

  1. 快照当前时间
     Timestamp now = Timestamp::now()

  2. 扫描 _lastActiveMap
     unique_lock(_connMutex):
       遍历 _lastActiveMap:
         if timeDifference(now, it->second) > 90.0:
           staleUsers.push_back(it->first)
           _lastActiveMap.erase(it)

  3. 清理每个过期用户
     for userid in staleUsers:
       ├── _userConnMap.find → shutdown() + erase
       ├── _redis.unsubscribe(userid)
       ├── _redis.delUserState(userid)
       ├── _userModel.updateState(offline)
       └── LOG_INFO "heartbeat timeout, user X disconnected"
```

### 10.4 PING/PONG 交互

```
客户端 (当前无主动 PING, 依赖聊天消息维持活跃)
  或: 手动发送 {msgid:11, id:userid}  → 服务端 ping()

服务端 ping():
  unique_lock: _lastActiveMap[userid] = time
  → 回复 {msgid:12} (PONG_MSG)

客户端 readTaskHandler:
  msgtype=12 (PONG_MSG) → continue (静默)
```

```

   │                               ├─ SET msg:1_xxx 1 NX EX 3600 →│

   │                               ├─ 跳过处理                     │
   │                               ├─ send ACK {status:"duplicate"}│
   │←── 收到 ACK {status:"duplicate"}│                             │
```

| 11 | PING_MSG | C→S | (手动/定时器) | ChatService::ping |
